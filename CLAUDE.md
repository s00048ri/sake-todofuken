# 清酒都道府県別生産量 長期パネルデータ構築＋可視化プロジェクト

## プロジェクト概要

日本の清酒（日本酒）の都道府県別生産量（製成数量）の長期時系列パネルデータを構築し、
歴史的な生産構造の変遷を可視化するプロジェクト。

既存の公開データセットは確認した限り存在せず、先駆的なデータセット構築となる。
完成後はGitHub等でオープンデータとして公開を検討。

## 検証したい仮説

1. **兵庫の一貫した強さ** — 灘五郷を中心に、江戸時代「下り酒」以来の圧倒的な生産基盤。
   白鶴・大関・菊正宗・日本盛など大手が集中。全期間を通じてシェア1位を維持しているか。

2. **淡麗辛口ブームでの新潟の台頭** — 1980〜90年代、越乃寒梅・八海山・久保田の人気爆発。
   新潟のランキング上昇がこの時期と重なるか。ブーム以前の水準との差も検証。

3. **近年の東北の台頭** — 山形（十四代、出羽桜）、秋田（新政、雪の茅舎）、福島（飛露喜、写楽）など
   全国新酒鑑評会での高評価。品質向上→ブランド力→生産量増加の流れがデータに現れるか。
   ※「量」より「単価・付加価値」の向上の可能性もあり、製成数量だけでは見えない仮説かもしれない。
   → **特定名称酒別データが鍵**：普通酒が減少する中で純米吟醸・純米酒の構成比が上昇しているか、
     都道府県ごとの特定名称酒比率の変化を追うことで「質の台頭」を定量的に検証できる。

## データの軸と粒度

### コアデータ: 清酒の都道府県別×年度別パネル
- 製成数量（総量）
- 課税移出数量（出荷ベース）

### 特定名称酒別データ（可能な範囲で）
特定名称酒制度は昭和63年（1988年）の「清酒の製法品質表示基準」告示で確立。
都道府県別×特定名称酒タイプ別のクロス集計が取れれば、仮説3の検証精度が大幅に上がる。

- **吟醸酒** — 精米歩合60%以下、吟醸造り
- **大吟醸酒** — 精米歩合50%以下、吟醸造り
- **純米酒** — 米・米麹のみ
- **純米吟醸酒** — 純米＋精米歩合60%以下
- **純米大吟醸酒** — 純米＋精米歩合50%以下
- **本醸造酒** — 精米歩合70%以下、醸造アルコール添加
- **特別純米酒** / **特別本醸造酒**
- **普通酒**（特定名称酒以外）

主なデータソース:
- 国税庁「清酒の製造状況等について」→ 都道府県別×タイプ別の製成数量（PDF）
- 国税庁「酒のしおり」付表 → 特定名称酒のタイプ別課税移出数量の推移（全国計、国税局別）
- 「清酒製造業の概況」→ 都道府県別の製成数量・課税移出数量推移

### 探索的に収集を検討する都道府県別酒類データ
以下は、データ探索の過程で都道府県別に取得可能と判明した場合に積極的に収集する。
すべてを網羅する必要はないが、清酒の文脈を豊かにするデータとして有用。

**生産・製造関連:**
- 酒類全体の品目別製成数量（焼酎・ビール・ワイン等との比較）
- 清酒製造免許場数（酒蔵数の推移）
- 製造能力・稼働率
- 酒造好適米の品種別生産量（農水省データ）

**流通・消費関連:**
- 都道府県別の酒類販売（消費）数量（品目別）
- 成人1人当たりの酒類消費量
- 家計調査の清酒支出額（県庁所在市別）

**経営・産業関連:**
- 清酒製造業の企業数推移（都道府県別）
- 出荷金額・単価の推移
- 企業タイプ別構成（大手 vs 地酒蔵）

**輸出関連:**
- 清酒の輸出数量・金額（国別、可能なら出荷元都道府県別）
- RESAS APIの輸出入データで酒類を品目コードで絞り込み

**品評会・品質関連:**
- 全国新酒鑑評会の金賞受賞蔵数（都道府県別、年度別）
  → 品質評価と生産量・特定名称酒比率との相関分析に有用

**進め方:** データ収集スクリプトを書く過程で、同じExcel/PDFの中に上記の関連テーブルが
含まれている場合は一緒にパースしておく。別途取得が必要なものは優先度を判断して追加。

## データソース

### 主要ソース

1. **国税庁統計年報（間接国税編・酒税）**
   - 「酒類製成及び手持高表」→「都道府県別製成数量」が核心データ
   - e-Stat上にExcel/PDFファイルとして年度別に公開
   - 明治9年の「第1回大蔵卿年報書」から続く歴史あるデータ
   - URL: https://www.nta.go.jp/publication/statistics/kokuzeicho/tokei_nenpo.htm
   - e-Stat: https://www.e-stat.go.jp/stat-search/files?page=1&toukei=00351010&tstat=000001043366

2. **国税庁「清酒の製造状況等について」**
   - 都道府県別×タイプ別（吟醸系・純米系・本醸造・普通酒）の詳細データ
   - 昭和53年（1978年）以降の体系的データ（昭和52年以前は参考値）
   - 製造数量は「アルコール分20度換算（kL）」
   - PDF形式で年度別に公開
   - URL: https://www.nta.go.jp/taxes/sake/shiori-gaikyo/seizojokyo/

3. **国税庁「酒のしおり」**
   - 全国合計の品目別課税数量・販売数量の推移（昭和期まで遡る時系列）
   - 都道府県別は「成人1人当たりの酒類販売（消費）数量表」のみ
   - Excel/PDF形式、毎年更新
   - URL: https://www.nta.go.jp/taxes/sake/shiori-gaikyo/shiori/

4. **国税庁「酒税課税関係等状況表」**
   - 都道府県別の販売（消費）数量等の状況表あり
   - 年度別にPDFで公開

### 補足ソース

- **日本酒造組合中央会** — 酒造場数、出荷量等の業界データ
  - URL: https://japansake.or.jp/sake/sake-statistics-data/
- **e-Stat API** — データベース化されている部分はAPI（REST/JSON/CSV）で取得可能
  - 要ユーザ登録＋アプリケーションID
  - URL: https://www.e-stat.go.jp/api/
- **国税庁「清酒製造業の概況」** — 都道府県別の企業数・製成数量推移・経営分析
  - 令和2年度以降は「酒類製造業及び酒類卸売業の概況」に統合
  - URL: https://www.nta.go.jp/taxes/sake/shiori-gaikyo/seishu/02.htm
- **農林水産省** — 酒造好適米（山田錦・五百万石等）の都道府県別生産量
  - 原料米側から生産構造の変化を裏付ける補足データとして有用

### APIについて

- **e-Stat API** — 国税庁統計年報の酒税関連データの大部分は「ファイル」形式での掲載であり、
  APIクエリ可能な「データベース」形式にはなっていない。API単体での自動取得は困難。
- **RESAS API** — 輸出入の貿易データが中心。国内の都道府県別生産量は扱っていない。本プロジェクトには不向き。
- 現実的にはExcel/PDFファイルのダウンロード＋パースが主な取得手段。

## データ収集戦略

### Phase 1: 近現代データ（平成期〜令和、約30年分）
- e-Stat上の国税庁統計年報Excelファイルを年度別にダウンロード
- URLパターンを特定し、スクリプトで一括取得
- openpyxlでパースし、都道府県×年度のパネルデータに整形
- **「清酒の製造状況等について」PDFから特定名称酒タイプ別×都道府県別データを優先的に取得**
  （tabula-pyまたはpdfplumber）
- 同一ファイル内にある関連テーブル（免許場数、企業数、出荷金額等）も可能な限り併せてパース
- 「酒のしおり」のExcelから全国計の長期推移データも取得（品目別課税数量・販売数量）

### Phase 2: 昭和後期データ（昭和40年代〜昭和末期）
- 国税庁サイト上のPDF/Excel（あれば）を取得
- 表レイアウトの年度間差異に注意しつつパース
- 清酒製造数量のピークである昭和48年（1973年）前後のデータが特に重要

### Phase 3: 昭和前期〜戦前データ（可能な範囲で）
- 国立国会図書館デジタルコレクション等を活用
- OCR + 手動補正が必要になる可能性大
- 優先度は低め。Phase 1-2の完成後に検討

### データ定義の注意点
- **製成数量** = 製造場で製成された数量（出荷とは異なる）
- **課税移出数量** = 製造場から移出（出荷）され課税対象となった数量 → 「出荷量」に近い
- **販売（消費）数量** = 各都道府県での販売量 → 消費地ベース（生産地ベースではない）
- 目的に応じて使い分けるが、「生産量の変遷」には製成数量または課税移出数量を使用
- 単位: kL（キロリットル）、一部データは20度換算

## アウトプット構成

### 1. バーチャートレース（時系列ランキングアニメーション）
- 各年度の都道府県別製成数量ランキングが入れ替わるアニメーション
- 新潟の台頭や全体の縮小傾向が直感的に伝わる
- 技術: D3.js or Flourish

### 2. 日本地図＋ダイナミックバブルアニメーション
- 都道府県の重心座標にバブルをプロット、サイズ＝製成数量
- 年度スライダーでアニメーション
- 灘・伏見への地理的集中度合い、東北の分散的台頭が視覚化
- 技術: D3.js + TopoJSON or Deck.gl

### 3. インタラクティブ探索サイト
- 都道府県選択（複数選択・比較）
- 期間フィルタ（スライダー）
- シェア推移と絶対量の切替表示
- 地域ブロック別集計（東北、関東、近畿、etc.）
- 上位N県の推移、特定県のドリルダウン
- **特定名称酒ビュー**: 都道府県ごとの特定名称酒構成比（積み上げ面グラフ）、
  普通酒 vs 特定名称酒の比率変化、純米系シフトの地域差
- **多次元探索**: 製成数量だけでなく、酒蔵数・出荷金額・鑑評会受賞数など
  取得できたデータを横断的に比較できるダッシュボード
- 技術: React + D3.js / Recharts

## 作業の進め方

1. **まずデータ構造の偵察** — e-Stat上の国税庁統計年報のExcelファイルを数年分ダウンロードし、
   表レイアウト（行列構成、ヘッダ位置、品目分類の粒度）を確認。
   同時に「清酒の製造状況等について」PDFの構造も偵察し、特定名称酒別データの所在を特定。
2. **パーサー開発** — レイアウトパターンを特定し、清酒×都道府県の製成数量を抽出するパーサーを作成。
   同じファイル内の関連テーブル（酒蔵数、出荷金額等）も発見次第パースに追加。
3. **パネルデータ構築** — 全年度分を統合し、欠損・異常値をチェック。
   コアデータ（製成数量）と拡張データ（特定名称酒別、酒蔵数等）を分離して管理。
4. **サンプル可視化** — 数年分でプロトタイプを作り、ビジュアルの方向性を確認
5. **本格可視化** — 全データでインタラクティブサイトを構築
6. **公開準備** — README、データ辞書、ライセンス整備

**データ探索の原則:** パーサー開発の過程で「こんなデータもあった」という発見があれば、
都度CLAUDE.mdに記録し、パースするかどうかを判断する。網羅性よりも、
仮説検証に役立つデータを優先的に取得する。

## メモ

- 清酒製造数量のピークは昭和48年（1973年）で177万kL
- 令和元年（2019年）にはピーク時の約1/4〜46万kLまで減少
- 昭和52年（1977年）以前のデータは調査体系が異なり「参考値」扱い
- 沖縄県は一部データで計上されていない場合がある

### 特定名称酒制度の沿革
- 昭和50年代〜: 「級別制度」（特級・一級・二級）で品質を分類
- 昭和63年（1988年）: 「清酒の製法品質表示基準」告示 → 特定名称酒8分類の確立
- 平成4年（1992年）: 級別制度の廃止
- したがって、特定名称酒別の統計は1990年前後以降しか存在しない
- それ以前は「級別」データが存在する可能性がある（特級≒吟醸、一級≒本醸造に近いが厳密には異なる）

### 都道府県別データで見えてくるかもしれないこと
- 兵庫・京都（灘・伏見）の大手集中 vs 地方の小規模多数の構造変化
- 地域ごとの「普通酒→特定名称酒シフト」の速度差
- 酒蔵数の減少と1蔵あたり生産量の変化（集約化の進展度合い）
- 消費地（東京・大阪）と生産地のギャップ
- 輸出拡大の恩恵を受けている県とそうでない県の差
